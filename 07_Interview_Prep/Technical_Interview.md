# 👾 Technical Interview (TR) Prep

The Technical Round (TR) tests your coding skills, core computer science concepts (OOPS, DBMS, OS), and your knowledge of the projects listed on your resume.

---

## 📊 Core CS Focus Areas

| Subject | Critical Topics | Sample Question |
| :--- | :--- | :--- |
| **DBMS & SQL** | Joins, Normalization, ACID properties, Indexes. | "What is the difference between primary key and unique key?" |
| **OOPS** | Polymorphism, Inheritance, Encapsulation, Abstraction. | "Explain Runtime Polymorphism with a code example." |
| **Core Languages** | Memory layouts, Pointers, Garbage Collection, JVM. | "How is memory managed in Python vs Java?" |
| **DSA** | Linked list loops, sorting complexities, hash maps. | "How do you detect a loop in a singly linked list?" |

---

## 💡 Standard TR Questions & Model Answers

### Q1: Tell me about your final year project.
* **Model Answer Frame**: Use the **STAR** method (Situation, Task, Action, Result).
  - *Situation*: "For our final year, we noticed that local businesses struggled to track inventory..."
  - *Task*: "My role was to design the database schema and build the backend API endpoints..."
  - *Action*: "I implemented indexing on frequently queried columns and set up Redis caching..."
  - *Result*: "This reduced query response times by 40% and successfully handled 100+ concurrent requests."

### Q2: What is normalization in databases?
* **Model Answer**: "Normalization is the process of organizing data in a database to reduce redundancy and improve data integrity. It involves dividing large tables into smaller ones and defining relationships between them. The standard normal forms are 1NF (atomic values), 2NF (remove partial dependency), and 3NF (remove transitive dependency)."
